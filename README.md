# Caravan Roguelite MVP (Unity, без внешних ассетов)

Полностью играбельный прототип "караванного" роглайта:
- карта из узлов с выбором маршрута;
- ресурсы каравана (HP, еда, золото, мораль, атака);
- события с выборами и последствиями;
- пошаговый простой бой;
- город/магазин;
- состояния победы/поражения;
- state machine и расширяемая модульная архитектура;
- вся графика генерируется кодом (UI + процедурные спрайты).

## 1) Структура

```text
Assets/
  Scripts/
    Core/        # state machine
    Data/        # модели карты и каравана
    Generation/  # процедурная генерация карты
    Events/      # библиотека событий
    Combat/      # библиотека врагов
    Gameplay/    # игровые стейты и bootstrap
    Map/         # визуализация карты
    UI/          # HUD, модальные окна, фабрика UI
Packages/
ProjectSettings/
```

## 2) Быстрый запуск

1. Установите Unity **2022.3 LTS**.
2. Откройте папку проекта в Unity Hub.
3. Создайте сцену `Assets/Scenes/Main.unity`.
4. Создайте пустой объект `GameRoot`.
5. Добавьте на него компонент `GameBootstrap` (`Assets/Scripts/Gameplay/GameBootstrap.cs`).
6. Добавьте сцену в Build Settings как Scene 0.
7. Нажмите Play.

> Камера и Canvas создаются автоматически, поэтому сцена может быть пустой кроме `GameRoot`.

## 3) Правила MVP

- На каждом переходе по карте тратится 1 еда.
- Если HP <= 0, мораль <= 0 или еда < 0 → поражение.
- Узлы:
  - **Event**: текст + 2 выбора;
  - **Combat**: обычный бой;
  - **City**: торговля/лечение/апгрейд атаки;
  - **Boss**: финальный бой, после победы — победа кампании.

## 4) State machine

`GameStateMachine` переключает состояния:
- `TravelState`
- `EventState`
- `CombatState`
- `CityState`
- `EndState`

Все игровые ветки выражены переходами стейтов через `ChangeState(...)`.

## 5) Как расширять

### Новые события
Файл: `Assets/Scripts/Events/EventLibrary.cs`
- Добавьте новый `EventDefinition` в список `CreateDefault()`.
- Каждый `EventChoice` задаёт изменение параметров: Gold/Food/Morale/Health/Attack.

### Новые враги
Файл: `Assets/Scripts/Combat/EnemyLibrary.cs`
- Добавьте новый `EnemyDefinition`.
- Последний элемент списка используется как босс.

### Новые биомы
Сейчас биомы представлены логикой генератора карты в `MapGenerator`.
Вариант расширения:
1. Добавить enum `BiomeType`.
2. Хранить биом в `MapNode`.
3. В `MapGenerator` выбирать биом по глубине/seed.
4. В `MapView` менять цвет узла и/или фон по биому.

### Новые апгрейды
- Быстрый путь: добавить новые пункты в `CityState`.
- Чистый путь: вынести покупки в отдельные классы (`IShopOffer`) и собрать их через библиотеку, как события и враги.

## 6) Принципы кода

- Маленькие классы с одной ролью.
- Ясные имена (по предметной области).
- Изолированные библиотеки контента (`EventLibrary`, `EnemyLibrary`).
- UI создаётся кодом (`UiFactory`, `ProceduralSpriteFactory`) — без внешних ассетов.
