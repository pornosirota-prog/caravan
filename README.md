# Caravan Roguelite MVP (Unity, без внешних ассетов)

Полностью играбельный прототип "караванного" роглайта:
- карта из узлов с выбором маршрута;
- ресурсы каравана (HP, еда, золото, мораль, атака);
- события с выборами и последствиями;
- пошаговый простой бой;
- город/магазин;
- состояния победы/поражения;
- state machine и расширяемая модульная архитектура;
- вся графика генерируется кодом (UI + процедурные спрайты).

## 1) Структура

```text
Assets/
  Scripts/
    Core/        # state machine
    Data/        # модели карты и каравана
    Generation/  # процедурная генерация карты
    Events/      # библиотека событий
    Combat/      # библиотека врагов
    Gameplay/    # игровые стейты и bootstrap
    Map/         # визуализация карты
    Strategy/    # новый стратегический режим (domain/services/ui)
    UI/          # HUD, модальные окна, фабрика UI
  Tests/
    EditMode/    # unit-тесты логики стратегического режима
Packages/
ProjectSettings/
```

## 2) Быстрый запуск

1. Установите Unity **2022.3 LTS**.
2. Откройте папку проекта в Unity Hub.
3. Создайте сцену `Assets/Scenes/Main.unity`.
4. Создайте пустой объект `GameRoot`.
5. Добавьте на него компонент `GameBootstrap` (`Assets/Scripts/Gameplay/GameBootstrap.cs`).
6. Добавьте сцену в Build Settings как Scene 0.
7. Нажмите Play.

> Камера и Canvas создаются автоматически, поэтому сцена может быть пустой кроме `GameRoot`.

## 3) Режимы игры

### Классический режим каравана
Работает как прежде через карту узлов (`TravelState`, `CombatState`, `CityState`, `EventState`).

### Новый стратегический режим
Открывается отдельным экраном через кнопку **"Стратегия"** в верхней панели HUD.

MVP-механика:
- сетка территорий 3x3;
- стартовая территория уже принадлежит игроку;
- атаковать можно только соседние нейтральные клетки;
- бой = серия математических примеров на время;
- правильные и быстрые ответы ускоряют захват;
- ошибки снижают прогресс и сбрасывают серию;
- при захвате начисляются золото/еда/очки;
- отображаются ресурсы, прогресс боя и история последних действий;
- кнопка **"К каравану"** возвращает в основной режим без рестарта сцены.

## 4) Архитектура стратегического режима

`Assets/Scripts/Strategy` разделён на независимые слои:

- `Domain/`
  - типы данных (`TerritoryModel`, `StrategyGameState`, `BattleSessionState`);
  - конфиг (`StrategyConfig`);
  - extension points через интерфейсы:
    - `IStrategyMapGenerator`
    - `IMathTaskGenerator`
    - `IBattleProgressionService`
    - `IRewardCalculator`
- `Services/`
  - `GridStrategyMapGenerator`
  - `AdditionMathTaskGenerator`
  - `ComboBattleProgressionService`
  - `TerritoryRewardCalculator`
  - `StrategyGameService` (оркестрация сессии боя, наград и обновления состояния)
- `UI/`
  - `StrategyScreen` (изолированный экран стратегического режима)

Интеграция в общий цикл сделана через отдельный `StrategyState`:
- классический режим не переписывался;
- состояние стратегического режима хранится отдельно внутри `StrategyScreen`;
- переключение режимов происходит через state machine.

## 5) Как расширять

### Extension points для будущих механик
Для добавления новых механик не требуется переписывать ядро экрана:
- новые типы территорий → `TerritoryType` + генератор карты;
- новые типы задач → реализация `IMathTaskGenerator`;
- новые правила боя / режимы боя → реализация `IBattleProgressionService`;
- новые формулы наград / ресурсы → реализация `IRewardCalculator`;
- дополнительные эффекты (TileEffect / Ability / Units / Buildings) — через расширение `TerritoryModel` и отдельные сервисы, подключаемые в `StrategyGameService`.

### Идеи следующего шага
- PvE-волны и асинхронные PvP-сессии.
- Сезонный рейтинг и дневные события.
- Разделение вычислений: перенос `StrategyGameService` на серверный слой.

## 6) Тесты

Добавлены EditMode unit-тесты (`Assets/Tests/EditMode/StrategyGameServiceTests.cs`):
- генерация стартовой карты;
- рост прогресса захвата;
- расчёт наград;
- проверка атакуемости соседних территорий;
- смена владельца территории после успешного боя.

Для запуска:
1. Откройте **Window → General → Test Runner**.
2. Выберите **EditMode**.
3. Запустите `StrategyGameServiceTests`.

## 7) Принципы кода

- Маленькие классы с одной ролью.
- Ясные имена (по предметной области).
- Изолированные библиотеки контента (`EventLibrary`, `EnemyLibrary`).
- UI создаётся кодом (`UiFactory`, `ProceduralSpriteFactory`) — без внешних ассетов.
- Новый режим добавлен как отдельный feature-модуль без опасного рефакторинга существующей механики.
